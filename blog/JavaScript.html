Ruby Hashes vs. JavaScript Objects

Today's Date: Saturday, February 13, 2016

Hi, todays blog entry is going to be on the Ruby hash compared to the Javascript object. A Ruby hash is a container object which is able to store a key, and that key's value in a combined pair. From my post on Hashes - > If we were to create a breakfast variable set equal to a new hash it would look some thing like this breakfast = { }. These curly braces signal a new hash. If we wanted to place some elements inside the hash we would asign a key element which is connected to a value element. So if we wanted to create a breakfast menu with food as the key and whether that food is tasty, healthy, or juicy we would set it up like this:





if we wanted to write the same program in JavaScript we would write this.

var breakfast = { bacon: "tasty", eggs: "tasty", oatmeal: "healthy", OJ: "juicy", }

At first they might look similar but you can actaully see that there are some key differences. For one, in JavaScript, defining variables must include the keyword var in front of the variable names. Also, within the object the name does not have to include quotation marks. Only the value of the name must include quotations if that value is a string.
If we wanted to access the value to one of the pairs inside the object, we must use the following sytax.

objectVariable.name will yield -> value var breakfast = { bacon: "tasty", eggs: "tasty", oatmeal: "healthy", OJ: "juicy", } console.log(breakfast.eggs) >> "tasty"

I hope this was simple and easy to understand. I hope now you understand a little more between objects in JavaScript and Hashes in Ruby.
